webpack初体验

一、前言

2015-12-03是公司的第一次前端交流会，印象深刻，同时也给了我警醒，不应该继续“浑浑噩噩”下去了。

模块化的工具之一便有webpack，故来接触学习，因听说seaJs对于现在来说算是一个过时品了(无奈…)， 我们应该赶上前端的潮流…

二、切入正题

1、基础概念

  • 安装webpack工具
    npm install webpack -g

  • 初始化package.json
    npm init

  • 将webpack保存至package
    npm install webpack --save-dev

  • 配置用到的loader
    打开web.config.js
    通过编辑resolveLoader.moduleTemplate字段来编辑Loader的名称优先级

  • 安装用到的loader
    npm install --save-dev xxx-loader

  • 使用loader
    三种方式：
    require声明显式调用
    尽量不要使用require的方式引用Loader，尤其是希望脚本不依赖环境（node.js或者浏览器）时。如果非要使用，可以使用!来将Loader和源码分开。所有的Loader的路径都是相对当前目录的相对路径。

    配置文件配置

    CLI配置

  • 查询参数
    Loader可以通过查询字符串（query string）来传递查询参数。查询字符串通过?或者url-loader?mimetype=image/png这样的格式添加到Loader上。

    注意：查询字符串的格式取决于Loader。大部分Loader接受普通的查询字符串格式（?key=value&ke2=value2），还有一些接受JSON对象格式的查询字符串（?{"key":"value","key2":"value2"}）。
