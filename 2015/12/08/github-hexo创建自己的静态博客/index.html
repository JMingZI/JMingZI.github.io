手把手教你github+hexo创建自己的静态博客

一、前言

用github+hexo搭建博客的好处：免费+markdown+速度还行（图片可以上传到七牛）
hexo 是一个静态博客程序框架，它可以将你用markdown写的md文件转化为静态的html文件，并且组合好后在github上展现，因为喜欢markdown文本格式，所以喜欢hexo。当然不止hexo，还有一个博客程序叫Jekyll，也不错。

二、安装

  • 安装hexo需要的环境
    nodejs
    git

  • 接着就可以安装hexo
    npm install -g hexo-cli

  • 通过hexo命令初始化自己的博客文件夹，比如：下面的myblog就是博客程序文件夹

  • 这样一个hexo博客程序就可以在本地跑起来了，在根目录myblog下打开cmd，按住shift同时点击鼠标右键可在菜单中看见在此处打开命令行窗口，接着输入
    hexo server
    你会看到命令行提示：

    打开浏览器，输入localhost:4000，惊喜呈现。。。

三、在github上创建自己的主页，github pages

  • 登录自己的github，点击右上角创建new repository，填入相应的内容，项目名为yourProjectName.github.io

  • 接着点击settings：

  • 接着点击页面下：你也可以看相应的内容说明：

  • 然后你就进入了编辑界面，这个界面就相当于你的主页，点击右下角continue

  • 接下来就是主题选择界面

  • 接着就ok了，选择分支，这里需要手动创建master分支，因为默认创建的是gh-pages分支，创建后切换到master分支，然后在新窗口中输入yourProjectName.github.io，就可以看到你自己的主页了。

  • 由于我自己已经创建过一个主页，github为每个人仅提供一个带域名的主页，所以我的会提示：

四、进入自己的博客文件夹，打开配置文件_config.yaml

  • 可以设置自己博客的相关信息，例如：
  • 那么我们现在需要设置的当然就是配置文件中的这几句：一般在配置文件最下面
  • 设置博客部署的方式，我们选择git，因为信赖，值得拥有。

五、发布文章（不带图片）

  • 在我们博客文件夹的根目录下
    hexo new "文章标题"

  • 然后打开myblog -> source -> _posts -> 你的文章.md文件，打开就可以直接用markdown编辑了。

  • 编辑完成后，
    hexo generate 生成静态文件
    hexo deploy 部署静态文件到你的github

  • 最后的最后，输入你的域名就可以看到了。
